Camera Configuration

Total camera count; directly impacts storage, bandwidth, and licensing requirements.
The pixel dimensions of video captured (e.g., 1080p, 4K). Higher resolutions provide more detail but require more resources.
The algorithm used to compress video data (e.g., H.264, H.265/HEVC, MJPEG). Modern codecs achieve better compression but may require more processing power.
The number of frames captured per second. Higher frame rates result in smoother motion but increase resource usage.
The amount of movement in the camera view. High motion can increase storage needs by 2-3x compared to static scenes.

Recording Configuration

Determines when video is recorded; continuous uses more storage while motion/event-based significantly reduces storage needs.
Number of days to keep recorded footage before deletion; directly proportional to required storage capacity.

Storage Configuration

Total raw storage capacity before RAID or formatting; larger drives provide more storage but may increase failure recovery time.
Determines data redundancy and performance characteristics; higher levels provide better protection but reduce usable capacity.
Reserved capacity to prevent storage saturation; recommended 10-20% buffer improves system performance and prevents unexpected outages.

Server Configuration

Maximum number of camera streams a single server can handle; limit depends on resolution, frame rate, and server specifications.
